第一章git概述

什么是版本控制：

  • 用于记录文件内容变化，以查阅不同版本的修订情况，可以记录文件修改的历史，并可以进行版本切换回滚

分布式版本控制与集中式版本控制：

  • 集中式：

    • cvs，svn，vss等具有一个集中管理的服务器，用于保存所有版本数据，每一个人都通过客户端同时在服务器对同一套代码进 行修改
    • 好处：看的是同一个项目，每个人都能看到所有人的修改，管理员也可以方便对每个人的管理
    • 坏处单点故障

  • 分布式

    • git，mecurial，darcs等分布式版本控制工具，没有中央服务器，每个个人电脑就是一个本地代码库，此外还有一个远程库以进行代码托管，并保证代码统一性
    • 流程：从远程克隆代码clone–修改代码–在本地仓库更新commit–推送到远程仓库push

git历史

git工作机制

  • 工作区:本地磁盘
    (磁盘上存放代码的位置，在工作区crud是没有历史记录的，就是实时的crud，不被记录）
    —git add—
    暂存区:相当于git的缓存，临时存储
    （使用该命令后，代码将被git所记-录追踪，被本地git临时记录的地方，可以删除，没有生成历史版本）
    —git commit—
    本地库
    （生成历史版本，不可删除修改，永远存在于历史版本中，除非删除本地库）
    —git push—
    远程库

  • 代码托管中心：基于网络服务器的远程代码仓库，比如github

    • 局域网(内部网)：gitlab
    • 互联网:github，gitee

##第三章 git常用命令

设置用户签名：

  • git config –global user.nagitme 用户名
  • git config –global user.email 邮箱
    打开.gitconfig文件查看以上用户配置信息（签名）是否成功（签名作用：区分操作者的身份）
    与github账号或其他代码托管中心没有关系，只是代表本地客户端的一个签名。

初始化本地库： git init

  • 要用git管理本地的目录，首先需要git获取该目录的管理权
  • 到指定目录下–在当前目录下打开git bash–使用git init命令初始化该目录–生成.git目录文件（不要修改）
  • ll -a : linux环境（git bash环境就是linux环境，因为git与linux是一个人写的，所以两者的命令是一致的）下查看隐藏文件的命令

查看本地状态: git status

  • 使用该命令后会输出一些log日志， 日志内容：
    • 默认分支
    • commit 历史纪录
    • 需要或可以commit的内容：
      红色文件：未被git追踪的内容，（即没有执行git add）
      绿色文件：被git追踪的内容，但没有被commit（即没有执行git commit，没有上传到本地库）

添加暂存区：git add 文件名

  • 删除暂存区中文件：git rm –cached 文件名（但在工作区中，该文件没有被删除，只是该文件不再被git追踪）

提交本地库：git commit -m “版本日志信息” 文件名

  • git commit -m “version 11.10” myfile.txt

    • 执行commit之后,会提示提交内容的变化, 以及本次提交的版本号

  • 执行 git commit后，git status状态

    • nothing to commit, working tree clean：当前工作区或暂存区中没有需要coommit的内容，工作树是干净的

  • 查看版本信息命令：git reflog
    例子：(HEAD->master) HEAD@{0}:commit (initial): first commit

    • 其中，965ca1是commit的版本号（后七位）
    • HEAD->master HEAD@{0}:commit (initial): first commit 表示当前所处版本
    • first commit 表示当时提交commit时，自己填写的版本日志信息

  • 查看详细版本信息命令：git log
    可以看到完整版的版本号，从commit的提交用户信息，提交日期，提交的版本日志信息

修改文件，体验版本迭代

  • 一但修改文件后，修改后的文件，git status将会是红色，即没有被git追踪
  • git add myfile.txt
  • git commit -m “second commit” myfile.txt
