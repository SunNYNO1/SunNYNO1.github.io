  1. 程序进程线程的基本概念

  • 程序(program)：
    一段静态的代码

  • 进程(process)：
    程序的一次执行,资源分配的最小单位

  • 线程(thread)：
    一个程序内部的一条执行路径，（进程中的不同功能即线程，项目或程序中一个main方法的执行即一个线程）
    调度和执行的最小单位
    共享进程的系统资源，但多个线程的资源共享会产生安全隐患（线程安全问题，使用线程同步解决该问题）

  • 单核与多核cpu的任务执行

    • 并行与并发
      并发：单核cpu同一时间段执行多个任务：“假的”多线程
      并行：多核cpu同同一时刻执行多个任务，真正的多线程
      一个java应用程序java.exe，至少三个线程：main主线程，gc垃圾回收线程，异常处理线程

  • 多线程的优点

    • 对于单核cpu多线程来说：
      （虽然 使用单个线程顺序执行多个任务，其实要比多个线程并发执行这些任务要快，因为后者需要线程间的切换）
      （1）用户体验好（多任务并发）：但 这种并发（假多线程），能够让用户在同一时间段进行多项任务，而不用再管理他们，而是让cpu在他们之间切换，让用户体验到并行的感觉
      （2）cpu利用率高（时间置换cpu利用率）：虽然cpu在多线程间不停切换，花了更多时间，但cpu不会出现空档期，使其被充分利用，而单个线程的顺序执行，在切换任务时，有可能出现cpu闲置的情况，导致cpu无法充分利用
      （3）程序结构改善：将冗长的进程，多个线程的分割，独立运行，并便于理解修改

  • 何时需要多线程

    • 同时执行多个任务
    • 等待任务：
      序列任务中，其中某一步子任务需要等待（例子：刷新网页时，图片比文字加载慢，使用多线程：可以先将文字加载出来，图片处空白；如果没有使用多线程：在图片加载完成前，整个网页都无法显示）
    • 后台任务

  1. 线程创建与使用
    2.1 继承Thread类

    package com.atguigu.java;
/**
* 多线程的创建，方式一：继承于Thread
* 1.创建Thread子类
* 2.重写run方法
* 3.创建子类对象
* 4。使用start方法启动线程
*
* 例子：遍历1000以内的所有偶数
*/
//继承Thread子类
class MyThread extends Thread{
    private int num;
    MyThread(int n){
        this.num = n;
    }
    //    重写run方法
    @Override
    public void run() {
        System.out.println("子线程开始");
        for(int i=0;i<=this.num;i++){
            if(i%2==0){
                System.out.println(i);
            }
        }
    }
}
public class ThreadTest {

    public static void main(String[] args) {
        System.out.println("开始主线程");
        //创建子类对象
        MyThread myThread = new MyThread(4);
        //使用start方法启动Thread：①启动当前线程 ；②调用线程的run方法
        myThread.start();
        System.out.println("回到主线程");
    }
}
