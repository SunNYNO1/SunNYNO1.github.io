方法（不同的实例需要训练不同的网络吗？？？？）

  • 概述

    • 给定一个语义分割图，输出目标实例可能出现的位置（借助STN学习仿射变换矩阵，仿射变换矩阵将一个1*1的单元bbox变换尺度并放置到分割图的指定位置中）,称为where模块

  • 给定位置（where的输出）与语义分割图，预测合理的实例形状（输出一个实例mask掩码），称为what模块

  • 将mask掩码按照where中的仿射变换矩阵    以指定尺度、位置放置到分割图中

  • where、what模块分别用来 学习语义分割图中 实例的空间分布（位置与尺度）、形状分布，基于条件GAN实现分布学习（每个模块由1个generator 与 2个discriminator）

  • 为减少训练过程中出现模式崩塌（即给出的位置总是在同一个位置，形状总是同一个形状），使用两种训练方式（无监督训练、监督训练），测试时只是用无监督过程

  • 无监督学习的输入X：语义分割图（where）或实例边界图（what） + 随即噪音

    监督学习输入X^+^：语义分割图或实例边界图 + 仿射变换矩阵A或实例mask掩码

    • 实例边界图：含有BBOX的语义分割图

    • 仿射变换矩阵A：从单元bbox到已知实例bbox进行变换得到的（其经过VAE编码得到一个向量ZA，该向量服从空间分布）

    • 实例Mask掩码：分割图中的已知实例掩码，其经过VAE编码为向量ZS，服从形状分布

    无监督输出（监督输出与之相同）：生成的仿射变换矩阵A^^^ （where） 或 生成的实例掩码（what）

    • 监督学习过程中，有映射关系：A—>ZA—>A^^^，已知mask—>ZS—>new mask ，监督学习的最终目的是使A与A^^^、已知mask 与 new mask尽可能地相似

    其中监督学习与非监督学习的generator是参数共享的

Augmentation for small object detection

摘要

在最近几年中，物体检测经历了突飞猛进的发展。 尽管有这些改进，但在检测大小物体之间的性能仍然存在很大差距。我们在挑战巨大的数据集MS COCO上分析了当前的最新模型Mask-RCNN。 我们表明，小物体的ground truth与预测锚之间的重叠远低于预期的IoU阈值。 我们推测这是由于两个因素造成的； （1）只有几个图像包含小物体，（2）即使在每个包含小物体的图像中，小物体也不会显得足够多。 因此，我们建议使用小物体对那些图像进行过采样，并通过多次复制粘贴小物体来增强每个图像。 我们评估了不同的粘贴增强策略，最终，我们在实例分割方面实现了9.7％的相对改进，在小型对象的目标检测上实现了7.1％的相对改进。 与MS COCO上的最新方法相比。

二、相关工作

对象检测：基于快速区域的卷积神经网络（Faster R-CNN）[32]，基于区域的完全卷积网络（R-FCN）[10]和SingleShot Detector（SSD）[26]是对象检测的三种主要方法，它们的区别在于 通过是否以及在何处附加区域提案[20]。 FasterR-CNN及其变体旨在帮助各种对象缩放，差异裁剪将所有建议合并为一个分辨率。 但是，这可能发生在深层卷积网络内部，并且所产生的裁剪框可能无法与对象完美对齐，这可能会损害其性能。在实践中，SSD扩展为反卷积单发检测器（DSSD）[15]，可以对低点进行高采样 通过在解码器部分[35]中进行转置卷积来对SSD的分辨率特征进行处理，以提高内部空间分辨率。类似地，特征金字塔网络（FPN）[24]扩展了带有解码器类型子网络的Faster R-CNN。

​ 实例分割：实例分割超出了对象检测范围，需要预测每个对象的确切遮罩。 多任务网络级联（MNC）[9]建立了预测和掩码细化的级联。 完全卷积实例感知语义分割（FCIS）[23]是一种完全卷积模型，可计算每个感兴趣区域共享的位置敏感分数图。[14]也是一种完全卷积方法，可学习像素嵌入。 MaskR-CNN [18]扩展了FPN模型的分支，用于预测蒙版，并引入了新的差异裁剪操作，用于对象检测和实例分割。

​ 小物体检测：小物体目标检测可以通过提高输入图像分辨率[7,26]或将高分辨率特征与低分辨率图像的高维特征融合[36,2,5,27]来解决。 但是，这种使用高分辨率的方法会增加计算开销，并且无法解决大型对象之间的不平衡问题。 [22]而是使用生成对抗网络（GAN）在卷积网络中构建在交通标志和行人检测的背景下在大小物体之间无法区分的特征。 [12]基于区域提议网络中的不同分辨率层使用不同的锚尺度。 [13]将图像特征按锚点大小的正确比例移动以覆盖它们之间的间隙。[6,33,8,19]在裁剪小对象建议时添加上下文。

    我们针对此超参数在1至15范围内的最佳值进行了研究。 我们针对所有实验测试了该范围内的每个值，发现切出前6个细心的贴片可以使我们在整个实验中获得最佳的平均性能。 此值的一种解释可能是，切出少于6个色块无法为第二张图像中的主要被摄对象提供足够的遮挡。 相反，切割6个以上的斑块可能会给图像中的原始对象提供过多的遮挡，从而对该图像的各个label不足以区分，使该模型难以学习任何有用的知识。

  5. 讨论

    ​ 我们的实验确实提供了有力的证据，表明我们的方法在不同数据集和体系结构上的平均效果优于CutMix和Mixup. Attentive Cut-Mix始终比其他方法平均提高1.5％，这证明了我们关注机制的有效性。 我们的方法的一个缺点是以下事实：除了要训练的实际网络之外，还需要预训练的特征提取器。 但是，根据分类任务以及模型和所用数据集的训练复杂性，我们可以更改用于数据增强的预训练提取器的大小。 因此，在整个训练方案中，这可以证明是微不足道的额外计算，并且很容易被该方法的性能增益所抵消。 我们认为，该方法可以为其他计算机视觉任务（如对象检测，实例分割等）提供类似的收益，因为它们都依赖于从图像中提取的鲁棒特征，这也是图像分类任务的主要基础。 因此，这项工作的未来方向将是将该方法用于其他视觉任务。
